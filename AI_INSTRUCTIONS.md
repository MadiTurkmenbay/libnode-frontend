# ИНСТРУКЦИЯ ДЛЯ AI АГЕНТОВ (Фронтенд - Nuxt 3)

## Описание
Фронтенд проекта LibNode. SSR-приложение, написанное на Vue 3 (Composition API) с использованием фреймворка Nuxt 3. Стилизация — Tailwind CSS + shadcn-vue.

## Структура папок
Не заставляй пользователя указывать папку, куда положить файл. Анализируй задачу и клади файлы строго сюда:

- `pages/` — Роутинг приложения на базе файловой системы (File-based routing). Каждый файл — это страница. Динамические роуты описываются в скобках `[id].vue`.
  - `login.vue` и `register.vue` — страницы авторизации (используют `useAuth()`).
  - `profile/` — папка защищенных маршрутов для профиля пользователя (включая закладки/списки `collections/`).
- `components/` — Vue-компоненты.
  - `ui/` — зарезервирована для базовых компонентов shadcn-vue (например, `Button`, `Input`, `Label` — некоторые созданы вручную).
  - `SiteHeader.vue` — глобальная шапка сайта (отвечает за навигацию и состояние авторизации, интегрирована в `app.vue`).
  - Остальные — доменные компоненты (например, `BookCard.vue`).
- `composables/` — Переиспользуемая логика (Composition API). Автоматически импортируются Nuxt.
  - `useAuth` — логика авторизации и состояние пользователя.
  - `useApiFetch` — кастомная обертка над `$fetch`/`useFetch` для автоматической подстановки JWT токенов из куки. Все запросы к API делать **строго через него**, а не через чистый `$fetch`!
- `middleware/` — Route guards. Защита маршрутов. Например, `auth.ts` предназначен для защиты роутов от неавторизованного доступа. Подключается явно через `definePageMeta`.
- `types/` — Глобальные TypeScript интерфейсы. Все модели данных здесь **обязаны зеркально отражать** DTO с бэкенда!
- `assets/` — Статика и глобальные стили (`globals.css`).
- `lib/` — Утилиты (например, обертки для shadcn, `utils.ts`).

## Основные паттерны проекта
1. **SSR (Server-Side Rendering)**: Мы используем SSR. Любой стейт, который должен быть сохранен между запросами (например, токены авторизации), хранится в куки (через `useCookie`). **ЗАПРЕЩАЕТСЯ** использовать `localStorage` на этапе инициализации приложения.
2. **Стилизация**: TailwindCSS. UI базируется на компонентах Radix Vue (устанавливаются через `shadcn-nuxt`).
3. **Состояние**: Не используем Pinia, если задача решается с помощью Composition API (`useState` + `readonly`). Пример — то, как реализован `useAuth`.
4. **Коллекции (закладки)**: Книга может находиться только в одной коллекции пользователя (взаимоисключающие коллекции). На странице книги `pages/books/[id].vue` используется радио-кнопочная логика: клик по коллекции, в которой книги нет — добавляет (удаляя из других), клик по уже выбранной — удаляет книгу из всех коллекций. Главная кнопка меняет текст и цвет на `secondary` при наличии книги в коллекции.
